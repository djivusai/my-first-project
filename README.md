## GitHub Concepts Guide

### Repositories
A repository is a folder for your project's files and version history.
- Create new: Click "+" icon â†’ New repository
- Public vs Private: Choose who can see your code
- Contains: Code, issues, pull requests, wiki

### Commits
A snapshot of your changes with a message explaining what was changed.
```bash
# Example: Making a commit locally
git add .
git commit -m "Add new feature"
git push origin main
```

### Branches
Separate lines of development to work on features safely.
- **main**: Production-ready code
- **feature branches**: Work on new features without affecting main
- Common naming: `feature/awesome-feature`, `bugfix/critical-bug`

### Pull Requests (PRs)
Propose changes from one branch to another. Used for code review.
1. Create a branch
2. Make changes and commit
3. Create a Pull Request
4. Team reviews and approves
5. Merge into main

### Issues
Track bugs, features, and tasks.
- Use labels: bug, feature, enhancement, documentation
- Link issues to PRs
- Assign to team members

## Useful Shortcuts
- `g + c`: Go to Code
- `g + i`: Go to Issues
- `g + p`: Go to Pull Requests
- `t`: Open file finder
- `?`: Show all keyboard shortcuts
